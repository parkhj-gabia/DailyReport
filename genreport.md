@template.txt을 템플릿으로 이용해서 @dailywork.txt를 적용해 dailyreport_데이터센터_YYYYMMDD.txt를 생성한다.
이때 변환규칙을 나열하고 이규칙을 윈도우즈의 파워쉘스크립트(@Generate-Report.ps1)로 작성한다.

규칙 
==== 
1. 헤더 형식
   - YYYY-MM-DD (요일) 야간근무자 - IDC운영유닛 <WorkerName>
   - YYYY-MM-DD는 실행일 기준으로 어제 날짜로 변환한다.
   - 야간근무자 이름(<WorkerName>)은 스크립트 실행 시 인자로 받는다.

2. 작업내역 (Group 2)
   - 서비스가 "하이웍스"가 아니고, 기재된 내용 중 "웹훅" 키워드가 포함되지 않은 모든 항목을 포함한다.
   - 형식: - [HH:MM] 회원아이디 / 아이피 - 처리내용
   - 시간순으로 정렬한다. (00:00~08:00는 전일 야간 근무로 간주하여 가장 뒤로 정렬)
   - 항목이 없는 경우 "- 없음" 출력

3. 장애처리 (Group 3)
   - 템플릿의 내용을 그대로 유지한다.

4. 83/가비아(웹훅)시스템지원 (Group 4)
   - 서비스가 **"하이웍스"**이거나, 처리내용에 **"웹훅"** 이라는 단어가 들어가는 모든 항목을 포함한다.
   - 형식: - [HH:MM] 처리내용
   - 항상 아래 두 줄을 상단에 추가한다.
     - [20:00~08:00] VOC 장애관련 문의 : 0건
     - [20:00~08:00] gabia.com 로그인 및 도메인 검색 기능 정상 확인
   - **데이터센터별 자동 추가 항목 (INTMON ARP 체크):**
     - 가산: 04:10(정상), 07:10(과천/가산 정상)
     - 서초: 07:10(서초/강남 정상)
   - 모든 항목을 시간순으로 정렬한다.

5. 공,클 수신메일 (Group 5)
   - 템플릿의 내용을 그대로 유지한다.

6. 육안점검 (Group 6)
   - 데이터센터에 따라 다르게 출력한다.
   - **서초:**
     - 6) 카버코리아 육안점검 특이사항 없음 메일 발송 완료
          씨젠 육안점검 특이사항 없음 메일 발송 완료
   - **가산:**
     - 6) 캘러웨이골프 코리아 특이사항
          - 육안점검 메일 발송완료

7. 기타 공통 사항
   - @dailywork.txt 형식: YYYY-MM-DD HH:MM 근무자 [Tab] 데이터 센터 [Tab] 그룹 [Tab] 서비스 [Tab] 회원아이디 [Tab] 아이피 [Tab] 처리내용 [Tab] [작업]
   - 데이터센터는 가산과 서초만 존재하며, 강남은 서초에 포함되고 과천은 가산에 포함된다.
   - 각 데이터센터(Center)별로 별도의 보고서 파일을 생성한다.
   - 야간근무자이름을 Benedict로 고정하는 설정은 제거하고 인자로 처리하도록 변경되었다.
