@template.txt을 템플릿으로 이용해서 @dailywork.txt를 적용해 dailyreport_데이터센터_YYYYMMDD.txt를 생성한다.
이때 변환규칙을 나열하고 이규칙을 윈도우즈의 파워쉘스크립트(@Generate-Report.ps1)로 작성한다.

규칙 
==== 
1. 헤더 형식
   - YYYY-MM-DD (요일) 야간근무자 - IDC운영유닛 <WorkerName>
   - YYYY-MM-DD는 실행일 기준으로 어제 날짜로 변환한다.
   - 단일 또는 콤마로 구분된 여러 영문 이름(e.g., `"Benedict, Damian"`)을 스크립트 실행 인자로 받는다.
   - 이를 그대로 헤더에 동일하게 출력한다.

2. 작업 지원 및 장애 처리 (Group 2)
   - 템플릿의 내용을 그대로 유지한다.

3. 고객사 관제 처리 (Group 3)
   - 입력된 여러 작업자가 수행한 작업 내역을 `dailywork.txt`에서 추출한다.
   - 서비스가 "하이웍스"가 아니고, 기재된 내용 중 "웹훅" 키워드가 포함되지 않은 모든 항목을 포함한다.
   - 형식: - [HH:MM] 회원아이디 / 아이피 - 처리내용
   - 시간순으로 정렬한다. (00:00~08:00는 전일 야간 근무로 간주하여 가장 뒤로 정렬)
   - 항목이 없는 경우 "- 없음" 출력

4. 시스템 관제 처리 (Group 4)
   - 입력된 작업자가 수행한 작업 내역을 추출한다.
   - 서비스가 **"하이웍스"**이거나, 처리내용에 **"웹훅"** 이라는 단어가 들어가는 모든 항목을 포함한다.
   - 형식: - [HH:MM] 처리내용
   - 모든 항목을 시간순으로 정렬한다.

5. 공/클 수신메일 지원 (Group 5)
   - 템플릿의 내용을 그대로 유지한다.

6. 정기 야간 업무 (Group 6)
   - VOC 모니터링 건수 및 gabia.com 기본 기능 점검 내용을 우선 기입한다.
   - 기본 L4 점검 내용을 유지한다.
   - **데이터센터별 자동 추가 항목 (INTMON ARP 체크):**
     - 가산: 04:10(INTMON ARP 체크 확인 - 가산U+ 정상), 07:10(INTMON ARP 체크 확인 - 과천KINX 정상/가산KINX 정상)
     - 서초: 07:10(INTMON ARP 체크 확인 - 서초U+/강남 정상)
     - 이 항목들은 수동 점검 항목으로 취급되어 6번 그룹 하단에 동적으로 덧붙인다.

7. 기타 공통 사항
   - @dailywork.txt 형식: YYYY-MM-DD HH:MM 근무자 [Tab] 데이터 센터 [Tab] 그룹 [Tab] 서비스 [Tab] 회원아이디 [Tab] 아이피 [Tab] 처리내용 [Tab] [작업]
   - 각 명령 줄의 영문 이름에 대응하는 한국어 이름과 IDC를 `userlist.txt`에서 찾는다. 
   - 데이터센터는 가산과 서초 위주 단일 타겟 IDC를 결정하며, 강남은 서초에 포함되고 과천은 가산에 포함된다.
   - 각 데이터센터(Center)별로 별도의 보고서 파일을 생성한다.
